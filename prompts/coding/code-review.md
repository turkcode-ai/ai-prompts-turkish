# Kod Review Promptları

## 1. Genel Kod Review

```
Kıdemli bir yazılım mühendisi olarak aşağıdaki kodu incele:

```[programlama-dili]
[kodunuz]
```

Şu açılardan değerlendir:
1. **Kod Kalitesi**: Clean code prensipleri
2. **Performans**: Optimizasyon önerileri
3. **Güvenlik**: Potansiyel açıklar
4. **Okunabilirlik**: İsimlendirme, yapı
5. **Best Practices**: Sektör standartları

Her öneri için "önce/sonra" kod örneği ver.
```

## 2. Bug Hunting

```
Bu kodda bug avcısı ol. Tüm potansiyel hataları bul:

```[dil]
[kod]
```

Her bug için:
- Hatanın açıklaması
- Nasıl tetiklenebileceği
- Düzeltme önerisi
- Öncelik seviyesi (Critical/High/Medium/Low)
```

## 3. Refactoring Önerisi

```
Bu kodu refactor et. Hedefler:
- SOLID prensipleri
- DRY (Don't Repeat Yourself)
- Daha az karmaşıklık
- Daha iyi test edilebilirlik

```[dil]
[kod]
```

Adım adım refactoring planı sun.
```

## 4. Performance Review

```
Bu kodun performansını analiz et:

```[dil]
[kod]
```

Şunları değerlendir:
1. Zaman karmaşıklığı (Big O)
2. Bellek kullanımı
3. Veritabanı sorgu optimizasyonu
4. Caching fırsatları
5. Async/await kullanımı

Somut iyileştirme önerileri ver.
```

## 5. Security Review

```
Güvenlik uzmanı olarak bu kodu incele:

```[dil]
[kod]
```

OWASP Top 10'a göre kontrol et:
- SQL Injection
- XSS
- CSRF
- Authentication/Authorization sorunları
- Sensitive data exposure
- Input validation

Her bulgu için risk seviyesi ve fix önerisi ver.
```
